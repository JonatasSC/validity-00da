# ============================================================================
# RESUMO DOS COMANDOS DESCOBERTOS
# ============================================================================
#
# Comandos básicos (1 byte):
#   0x01 - ROM info (retorna 38 bytes com estado no último byte)
#   0x19 - Query state (retorna 64+4 bytes com estado detalhado)
#
# Comando 0x8e (17 bytes, formato: 8e SS 00 02 00 00 00 00 00 00 00 00 00 00 00 00 00):
#   0x8e 0x09 - Read sensor info (26 bytes resposta)
#   0x8e 0x1a - Read sensor config/calibração (78 bytes)
#   0x8e 0x2e - Read calibration blob (3586 bytes - parâmetros floating point)
#   0x8e 0x2f - Read firmware version (18 bytes)
#
# TLS Handshake:
#   - Header proprietário de 4 bytes (44 00 00 00) antes dos records TLS do HOST
#   - Cipher suite selecionada: TLS_ECDH_ECDSA_WITH_AES_256_GCM_SHA384 (0xc02e)
#   - Certificado: X.509 com chave ECDSA (secp256r1)
#   - Sem Server Certificate Request (sensor não pede cert do host)
#
# Sequência de provisioning completa:
#   1. MSG1 (0x01) → verifica estado
#   2. 0x8e 0x09 → lê info do sensor
#   3. 0x8e 0x1a → lê config/calibração
#   4. 0x8e 0x2e → lê blob de calibração (3586 bytes)
#   5. 0x8e 0x2f → lê versão firmware
#   6. MSG2 (0x19) → query state detalhado
#   7. [Repete passos 1-6 uma segunda vez]
#   8. TLS ClientHello (com header 44000000)
#   9. TLS ServerHello → cipher c02e
#   10. Certificate + ChangeCipherSpec + Finished
#   11. Provisioning via TLS (comandos criptografados)
#   12. Enrollment (leitura de digitais via TLS)
