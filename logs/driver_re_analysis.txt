 FAMÍLIA: TUDOR (nome interno do sensor)

Prefixo das funções: tudor*, ssi*, pal*, vfm*

═══════════════════════════════════════════════════════════════
 FLUXO TLS (reconstruído a partir de debug strings)
═══════════════════════════════════════════════════════════════

1. tudorTlsInitCtx() - inicializa contexto TLS
2. ssiTlsInit() - inicializa SSI TLS
   - "Only tlsSecurityTypeEcc and tlsSecurityTypePsk are supported"
3. tudorTlsEstablishSession()
   → ssiTlsEstablish()
   → "BEGIN TLS"
   → "STEP: TlsPrepareValidate"
   → "STEP: Step 1 of TLS Handshake process" (ClientHello)
   → "STEP: Step 2 of TLS Handshake process" (resposta)
   → Se erro: "STEP: TLS Alert generated during the handshake"
   → Se ok: "STEP: TLS handshake done!"
   → "END TLS"

4. Após TLS:
   - ssiTlsWrap() / ssiTlsUnwrap() - encripta/decripta dados
   - tudorTlsDataGet() / tudorTlsDataRestore()

═══════════════════════════════════════════════════════════════
 PAIRING / OWNERSHIP (key exchange)
═══════════════════════════════════════════════════════════════

Fluxo de pairing:
1. tudorSecurityGetSSPubKey() - obtém SS (Sensor Secret) Public Key
   - "SS Public Key not found" se não existir
   - A SSPubKey vem do sensor e é FIXA no firmware

2. _tudorSecurityGenHostKeyPair()
   - palGenHSPrivKey()
   - palCryptEccEllipticCurveSet()
   - palCryptoEccKeypairGenerate()
   - palCryptoEccExportPublicKey()
   - palCryptoEccExportPrivateKey()
   → Gera par de chaves ECC host (efêmero)

3. _tudorSecuritySignHPubK()
   → Assina a chave pública do host

4. _tudorSecuritySendPairingCommand()
   → VCSFW_CMD_TAKE_OWNERSHIP_EX2
   → Envia chave pública assinada + certificado pro sensor

5. _tudorSecuritySensorCertificateValidate()
   → tudorSecurityGetSSPubKey()
   → palCryptoEccCreatePublicKeyHandle()
   → palCryptoECDSAVerify()
   → Valida certificado do sensor com SSPubKey

6. Resultado salvo como "pairing data"
   - _tudorSecuritySetPairingData()
   - tudorSecurityGetPairingData()
   - _wrapPairingData() / _unwrapPairingData()

═══════════════════════════════════════════════════════════════
 TLS HANDSHAKE (detalhes internos)
═══════════════════════════════════════════════════════════════

Estruturas TLS:
- pTLSClntCtx - contexto do cliente TLS
- pTLSClntCtx->TLSCtx.cipher_specp - cipher spec
- pTLSClntCtx->TLSCtx.server_certp - certificado do servidor
- pTLSClntCtx->TLSCtx.pskp - PSK (pre-shared key)
- pTLSCtx->client_randomp - random do cliente
- pTLSCtx->server_randomp - random do servidor
- pTLSCtx->pEncHPubkey - chave pública do host encriptada

PRF (Pseudo Random Function):
- prfTLS12() - TLS 1.2 PRF
- prfAesCmac() - AES-CMAC PRF (alternativo)
- "master secret" - label para derivar master secret
- "key expansion" - label para derivar chaves de sessão

Key Exchange:
- pPreMasterSecret → derivado de ECDH
- ECDH entre chave efêmera do host e chave pública do sensor
- "Failed to import Server's public key" → sensor envia pubkey
- "Failed to export ephemeral ECC public key" → host gera efêmera
- "ECDH shared key generation failed" → shared secret = PMS

Cripto implementada via:
- BCrypt (Windows CNG): ECDH, ECDSA, AES
- IPP (Intel): SHA256, SHA384, AES-GCM, AES-CBC, ECC

═══════════════════════════════════════════════════════════════
 INSIGHT CRUCIAL: SS PUBLIC KEY
═══════════════════════════════════════════════════════════════

A "SS Public Key" (Sensor Secret Public Key) é a chave que:
1. Está gravada no firmware do sensor
2. É usada para VALIDAR o certificado do sensor
3. É usada como base para o ECDH (pre-master secret)

O driver obtém ela via tudorSecurityGetSSPubKey()
Se não encontrada: "SS Public Key not found" → pairing falha

ESTA é a chave que precisamos encontrar no firmware ou
na captura de tráfego para resolver o ECDH!
